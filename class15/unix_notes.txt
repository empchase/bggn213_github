## Here are the commands we've leared today in bash:
- git : runs git commands
- pwd : print working directory, how to print out where you "are" in your file system. Gives absolute path
- cd : change directory, give it the relative path w/r to your current working directory, or an absolute path
- ls : list, lists out the files/subdirectories in your current directory or in a specified file path
- mkdir: make a directory (default argument is at the current working directory)
- rm : removes everything in your directory downward if no argument. use it with an argument :)
- nano : open up a text editor. give it a (new or not) file name to open/start a document to write on. 
- cp : copy a file (cp source destination)
- mv : rename or move a file (mv source destination)
- less : to see the contents of a text file
- whoami : tells you the computer username 
- less : see the scrollable contents of text file
- head : view the top lines of a file
- grep : pattern match


- absolute path : where stuff is, spelled out from the root of the file systems (eg /Users/emilychase/Desktop/phd/bggn213/class15)
- relative path : where stuff is relative to where you are currently (ie your current working directory)

- ~ : home directory
- . : current directory
- .. : parent directory
- * : wild card (matches all characters)

### Process control
_ ^c : kill
- ^z : sleep
- fg : put the sleeping thing to run in the foreground
- bg : put the sleeping thing to run in the background
- top : display the top running processes in real time
- ps : display the running commands
- kill <PID> : kill a job with this PID
